大纲：
用于测试活动的，开发套件。

第一章，样例章节。
引子，从一个测试需求开始开始，引出测试方法、测试环境、自动化对接、测试用例编写等。

测试需求及测试用例，向量加法：
输入1，长度32个数，int32类型，每个数4字节，共32*4=128字节。
输入2，长度32个数，int32类型，每个数4字节，共32*4=128字节。
数据处理：两组数，对应位置，相加。
输出1：长度32个数，int32类型，每个数4字节，共32*4=128字节。

广义测试对象（芯片系统，驱动软件接口，web服务接口等）
具体测试对象（待测接口，待测API，具体提供功能的待测对象。例如：
ISA指令集，使用汇编编写用例；
驱动接口，使用c，c++编写用例；
网络服务，使用python，flask构造web服务，使用python构造用例。

通过样例引出，若待测接口有多个，待测参数有多个，待测取值范围有多个，怎样实现批量用例测试等。

哪些属于测试用例编写层级。
哪些属于测试自动化对接层级（例如对接ISA编译器等、自动化执行用例、测试报告生成等）。


第二章，自动化测试框架设计，包含数据生成、用例生成、测试环境对接层、测试报告生成等。


第三章，工具章节。

3.1 测试数据自动生成工具
数据格式（fp32，fp16，bf16，int8等）
数据个数，
数据取值范围（正数，从0到1，间隔，等等），
数据形状（例如8*32，一维，等等）
输出文件格式（十进制文本，十六进制文本，二进制bin）。

3.2 测试用例文件自动生成工具
用例文件模板，用例参数名及取值通过文档表格（例如excel）配置（支持自动遍历组合等），用例文件自动生成。

3.3 测试环境对接接口预留。
批量用例并发测试。
多服务器集群多任务测试环境。
远程环境测试。
多个远程环境并发、协同测试。（例如面向riscv的多线程环境gdb+openocd+fpga/spike。）

3.4 测试报告自动化生成。
批量用例测试结果分析。
自动化生成测试报告。
